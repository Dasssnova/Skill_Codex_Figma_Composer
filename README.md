# Figma Composer

`figma-compozer` превращает UI-референсы в Figma в структурированные и редактируемые макеты.

## Как подготовить Figma-файл

1. Поместите каждый референс отдельным изображением внутрь собственного Figma frame.
2. Размер frame может быть любым: навык использует его ширину и высоту как размер итогового макета.
3. Настройте у frame нужные скругления, Corner smoothing и clipping. Эти свойства будут повторены на выходе.
4. Названия frame и слоя изображения могут быть любыми. Они не используются для поиска или анализа референса.
5. Для нескольких экранов создайте отдельный frame для каждого референса.
6. При необходимости добавьте любые фотографии и растровые изображения в `figma-compozer/assets/`. Навык не подбирает их по смыслу: он распределяет разные доступные файлы по медиаконтейнерам.
7. Передайте Codex ссылку на файл, нужный node или выделенную область с референсами.

Не помещайте в область обработки посторонние изображения, иллюстрации и элементы, которые не являются UI-референсами.

## Запуск

```text
Используй $figma-compozer для этой Figma-ссылки: <ссылка>
```

## Что делает навык

Навык независимо анализирует каждый референс и создает рядом новый редактируемый экран. Он:

- повторяет размер, скругления, Corner smoothing и clipping исходного frame;
- восстанавливает композицию, размеры, отступы и иерархию слоев;
- собирает карточки, кнопки, поля, навигацию, меню, панели и другие логические блоки через Figma Auto Layout;
- отдельно ищет типографику и шрифтовые пары для каждого референса среди доступных Figma-библиотек, локальных, системных и Google Fonts, сравнивая основной вариант с тремя правдоподобными альтернативами и проверяя кириллицу;
- воспроизводит цвета, градиенты, обводки, прозрачность, тени и размытие нативными свойствами Figma;
- сохраняет повороты, перекрытия и обрезку элементов;
- создает редактируемые текстовые слои, фигуры и контейнеры;
- отделяет иконки от их подложек и использует только редактируемые векторные иконки из Figma-библиотек, текущего файла или Community;
- берет любые разные фото и изображения из `figma-compozer/assets/`, если они доступны;
- определяет непрерывные скругления и применяет нативный Corner smoothing;
- определяет прозрачные слои и воспроизводит их через нативные Opacity-настройки Figma;
- никогда не использует Unicode-символы, emoji, icon fonts, symbol fonts или другие текстовые глифы вместо иконок;
- заменяет системную клавиатуру и недоступные растровые медиа редактируемыми плейсхолдерами с вертикальным градиентом `#D8E0EA` → `#BAC6D7` и opacity `58%`.

Навык не объединяет стили разных референсов и не использует название устройства или frame для определения размеров результата.

## Как получить максимальное качество

Для сложных интерфейсов передавайте один референс за запуск. Это лучший режим для точной типографики, сложных градиентов, прозрачности и большого количества мелких элементов.

Допустимые размеры задачи:

- один сложный референс — рекомендуемый вариант;
- два или три простых референса можно передать вместе;
- большую подборку можно передать одной ссылкой, но навык всё равно должен завершать экраны последовательно, по одному.

Навык сначала обнаруживает все исходные frame и определяет порядок экранов. Затем для каждого референса отдельно:

1. Анализирует размеры, структуру, типографику, цвета, градиенты и прозрачность.
2. Создаёт корневой frame и каркас крупных областей.
3. Собирает интерфейс небольшими блоками: header, карточки, формы, списки, toolbar и navigation.
4. Выполняет не более 10 логических Figma-операций за один технический вызов.
5. После каждого крупного блока делает screenshot-проверку и сразу исправляет расхождения.
6. Проверяет весь готовый экран в масштабе исходного референса.
7. Переходит к следующему референсу только после завершения текущего.

Не просите навык построить все референсы одним большим Figma-вызовом. Большие монолитные операции хуже сохраняют иерархию, сложнее проверяются и чаще накапливают ошибки геометрии, текста, градиентов и opacity.

Для нескольких референсов используйте такой запрос:

```text
Используй $figma-compozer для этой Figma-ссылки: <ссылка>.
Обрабатывай референсы последовательно: полностью построй и проверь один экран, затем переходи к следующему.
```
